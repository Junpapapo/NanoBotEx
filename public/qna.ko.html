NanoBot AI를 최상의 상태로 구동하고 사용하는 팁을 모아두었습니다.

Q Gemini Nano 로컬 AI를 사용하기 위한 크롬 환경 설정은 어떻게 하나요?

Chrome의 내장 AI(Gemini Nano) API를 활성화하기 위해 아래 5단계를 수행해야 합니다.

🔧 크롬 빌트인 AI 활성화 단계:
  1. 크롬 주소창에 chrome://flags를 입력하고 엔터를 누릅니다.
  2. 상단 검색창에 prompt를 입력하여 Prompt API for Gemini Nano를 찾은 후 값을 Enabled로 변경합니다.
  3. 이어서 검색창에 optimization을 입력하여 Enables optimization guide on-device model를 찾은 후 값을 Enabled BypassPerfRequirement 또는 Enabled로 변경합니다.
  4. 우측 하단의 [Relaunch] 버튼을 클릭해 크롬 브라우저를 재시작합니다.
  5. 마지막으로 chrome://components 페이지로 이동하여 Optimization Guide On Device Model을 찾고 [Check for update](업데이트 확인) 버튼을 눌러 AI 모델 다운로드가 100% 완료될 때까지 잠시 대기합니다.
🤖 NanoBot: "크롬 브라우저의 소규모 뇌 이식 수술입니다! 절차가 다소 생소할 수 있지만, 한 번 세팅해두면 무한한 로컬 대화가 가능해집니다."
Q NanoBot은 대화 내역이나 사용 기록을 외부 서버로 전송하나요?

아닙니다. NanoBot은 100% 프라이버시가 보장되는 온디바이스 형태로만 동작합니다.

당사는 사용자의 대화 로그, 챗봇 이름, 커스텀 프롬프트 지침 등 어떠한 민감 정보도 클라우드나 외부 기업 서버로 전송하거나 보관하지 않습니다. 모든 자료는 사용자의 웹 브라우저 로컬 저장소(IndexedDB / LocalStorage) 내에만 온전히 보관됩니다.

🤖 NanoBot: "사용자의 모든 기밀 사항은 안전 격리실(로컬 저장소)에 영원히 격리됩니다. 인터넷이 끊기거나 지구가 멸망해도 제 연산은 계속됩니다!"
Q 프리미엄 기능인 버디(Buddy)는 어떤 역할을 하나요? Premium

AI 버디(Buddy)는 특별한 페르소나와 작업 지시사항을 부여받은 나만의 맞춤형 비서입니다.

기본 AI 대화 모드 외에, 텍스트 번역 전문가, 소스코드 리뷰어, 이메일 보정 비서 등의 역할을 미리 프로그래밍하여 다수의 버디를 생성하고 저장할 수 있습니다. 필요할 때마다 상단의 버디 목록에서 대화 상대를 스위칭하여 특정 전문 분야에 특화된 연산을 즉시 수행할 수 있습니다.

🤖 NanoBot: "작업 용도별로 다양한 분신을 호출할 수 있는 마법의 아이템입니다! 단, 분신들끼리 서로 수다를 떨게 하지는 마세요."
Q 나노봇의 아바타와 봇의 이름은 어떻게 변경하나요?

확장 프로그램 팝업창을 열면 좌측에 아바타와 봇 이름 세팅 영역이 있습니다.

  • 좌우 화살표를 눌러 아바타 일러스트를 스크롤하며 마음에 드는 아바타를 찾습니다.
  • 그 아래 봇의 이름(기본값: Name) 란에 원하는 타이틀을 기입하시면 실시간으로 비서의 이름이 설정 및 연동됩니다.
  • 셔플(Shuffle) 단추를 누르면 다채로운 아바타 카드 중 무작위로 아바타를 선택해 줍니다.
🤖 NanoBot: "얼굴도 이름도 사용자 취향대로 뜯어(?)고칠 수 있습니다. 단, 밥 늦게 주는 이름은 곤란해요!"
Q 로컬 AI 모델 세션이 제대로 잡혀있는지 어떻게 수동으로 확인하나요?

팝업 상단 헤더 영역에 있는 [AI ON] 버튼을 클릭하십시오. 즉각적으로 브라우저가 로컬 AI 엔진의 API 통신을 진단합니다.

  • TEST OK: 로컬 엔진 세션이 정상 구동되어 대화할 준비가 완벽히 되었음을 뜻합니다.
  • TEST FAIL: Chrome 환경 설정(Flags) 또는 컴포넌트 모델 파일의 백그라운드 다운로드가 덜 끝났을 가능성이 있습니다. 상단의 '환경 설정 가이드'를 다시 한 번 점검해 주십시오.
🤖 NanoBot: "저의 기동 스위치 같은 버튼입니다. 초록색으로 불이 켜진다면 뇌파 동조율 100%!"
